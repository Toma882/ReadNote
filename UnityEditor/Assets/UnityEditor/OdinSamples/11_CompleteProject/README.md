# 完整项目示例：游戏数据管理器

## 📖 项目概述

这是一个完整的 Odin Inspector 应用示例，展示了如何构建一个专业的游戏数据编辑器。

本项目参考了官方的 RPG Editor，但更简化、注释更详细，适合学习和参考。

## 🎯 学习目标

通过这个完整示例，你将学会：

1. **数据驱动设计** - 使用 ScriptableObject 管理游戏数据
2. **自定义编辑器窗口** - 使用 OdinMenuEditorWindow 构建专业工具
3. **模块化架构** - 如何组织大型编辑器项目
4. **实际工作流程** - 创建、编辑、管理游戏数据的完整流程
5. **高级 Odin 技巧** - 属性组合、自定义绘制、拖放等

## 🚀 使用方法

### 打开编辑器

在 Unity 菜单中选择：
```
Tools > Odin Samples > Complete Project > 打开游戏数据编辑器
```

### 创建数据

1. **创建角色**：点击工具栏的"创建角色"按钮
2. **创建技能**：点击工具栏的"创建技能"按钮
3. **创建物品**：点击工具栏的"创建物品"按钮

### 编辑数据

在左侧菜单树中选择要编辑的数据，右侧会显示详细的属性编辑器。

### 管理数据

- 使用搜索栏快速查找数据
- 使用分类菜单浏览不同类型的数据
- 支持拖放操作（如将技能拖到角色上）

## 💡 核心设计思路

### 1. 数据驱动架构

所有游戏数据都存储为 ScriptableObject 资源：
- **优势**：数据与逻辑分离，策划可以独立调整参数
- **实现**：继承 SerializedScriptableObject 以使用 Odin 的强大序列化

### 2. 模块化设计

系统分为角色、技能、物品三个独立模块：
- 每个模块可以独立开发和测试
- 模块间通过接口和引用关联
- 易于扩展新的数据类型

### 3. 用户友好的编辑器

使用 Odin Inspector 的各种特性创建直观的编辑界面：
- **分组布局**：BoxGroup、TabGroup、FoldoutGroup
- **数据验证**：Required、ValidateInput
- **自定义绘制**：PreviewField、TableList
- **交互功能**：Button、OnValueChanged

### 4. 灵活的属性系统

通过 StatList 实现灵活的属性管理：
- 支持动态添加/删除属性
- 类型安全的属性访问
- 编辑器中直观的显示
