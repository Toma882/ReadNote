# 系统层次设计

> **说明**：本文档描述的是**策划设计层面的系统分层**，关注游戏设计内容与机制的逻辑关系，用于指导策划进行系统设计，明确设计优先级和系统间的依赖关系。

## 系统层次关系

从策划设计层次来看，系统分为四层：

### 0. 概念层级（Concept Layer）

- 游戏的核心概念元素和世界观基础
- 定义游戏的文化背景、哲学理念、世界观设定
- 为所有系统提供概念基础和设计方向
- 是最抽象的层级，不涉及具体机制，只定义核心概念
- **示例**：
  - **中国玄学体系**：五行理论、阴阳理论、八卦理论、奇门遁甲等
  - **道教神明体系**：三清、四御、五方五老、六丁六甲、二十八宿等
  - **传统文化概念**：因果报应、功德业障、修炼体系、天人合一等
  - **世界观设定**：人鬼神三界、天地人三才、时空概念等

**作用**：
- 为顶级系统提供概念基础（如：战斗系统基于五行相生相克）
- 为支撑系统提供文化背景（如：装备系统体现道教法器概念）
- 为应用系统提供设计灵感（如：道具合成体现炼丹炼器概念）

### 1. 顶级层级（Top-level Layer）

- 核心概念系统设计
- 定义游戏的核心玩法和设计理念
- 直接与玩家行为交互，形成完整的游戏循环
- **示例**：战斗系统、任务系统、探索系统、社交系统、信仰系统、功德系统、混沌世界系统

### 2. 支撑层级（Supporting Layer）

- 为顶级系统提供设计层面的支撑子系统
- 是顶级系统运行和表达的基础内容系统
- 属于顶级系统的子层级机制，支撑顶级系统的功能实现
- **示例**：道具系统、装备系统、技能系统、剧情系统、对话系统等

### 3. 应用层级（Application Layer）

- 在支撑系统基础上的具体应用机制
- 将支撑系统的内容进行组合、加工、应用
- 是支撑系统的功能扩展和应用层
- **示例**：道具交易，道具合成，装备强化，技能系统，对话分支系统等

## 分层逻辑

- **概念层级** → 定义"是什么"（世界观、文化背景、核心概念）
- **顶级层级** → 定义"做什么"（核心玩法）
- **支撑层级** → 定义"用什么"（内容支撑）
- **应用层级** → 定义"怎么用"（应用机制）

**层级关系**：
```
概念层级（世界观、文化背景）
    ↓ 提供概念基础
顶级层级（核心玩法）
    ↓ 提供内容支撑
支撑层级（内容系统）
    ↓ 提供应用机制
应用层级（玩法扩展）
```

## 系统间关系说明

### 系统间的相互影响

**重要原则**：系统之间是**相互影响**的关系，不是单向的层级关系。

1. **顶级系统之间的相互影响**
   - 顶级系统之间会相互影响，形成复杂的交互网络
   - 例如：战斗系统影响任务系统（战斗完成触发任务），任务系统影响探索系统（任务引导探索），探索系统影响战斗系统（探索发现战斗）
   - 这种相互影响是游戏设计的重要特征，让游戏世界更加有机和动态

2. **支撑系统的跨系统特性**
   - 一个支撑系统可能同时服务于多个顶级系统
   - 例如：对话系统同时服务于任务系统、探索系统、社交系统
   - 例如：道具系统同时服务于战斗系统、探索系统、任务系统
   - 这种跨系统特性让支撑系统成为系统间连接的桥梁

3. **应用系统的组合特性**
   - 一个应用系统可能基于多个支撑系统
   - 例如：装备强化系统可能基于装备系统和道具系统
   - 这种组合特性让应用系统能够创造更丰富的玩法

### 关系示例

```
顶级系统之间的相互影响：
  战斗系统 ↔ 任务系统 ↔ 探索系统 ↔ 社交系统
  （相互影响，形成闭环）

支撑系统的跨系统服务：
  对话系统 → 任务系统、探索系统、社交系统
  道具系统 → 战斗系统、探索系统、任务系统
  剧情系统 → 任务系统、探索系统、社交系统

应用系统的组合特性：
  装备强化系统 → 基于装备系统 + 道具系统
  技能组合系统 → 基于技能系统 + 道具系统
```

## 设计原则

1. **层次清晰**：从概念到内容再到应用，层次分明
2. **符合策划视角**：关注设计内容，而非技术实现
3. **易于扩展**：每个层级都可以独立扩展
4. **逻辑合理**：支撑系统支撑顶级系统，应用系统扩展支撑系统
5. **相互影响**：系统之间相互影响，形成有机的游戏世界

## 对策划设计的指导价值

### ✅ 设计思路清晰

这个分层设计为策划提供了清晰的设计思路：

1. **自上而下的设计流程**
   ```
   第零步：确定概念层级（世界观、文化背景、核心概念）
   ↓
   第一步：确定顶级系统（核心玩法，基于概念层级）
   ↓
   第二步：设计支撑系统（内容支撑，体现概念层级）
   ↓
   第三步：设计应用系统（应用机制，应用概念层级）
   ```

2. **设计优先级明确**
   - **最优先：概念层级**：确定游戏的世界观和文化背景，为所有系统提供概念基础
   - **优先设计顶级系统**：确保核心玩法完整，基于概念层级设计
   - **其次设计支撑系统**：确保有足够的内容支撑，体现概念层级
   - **最后设计应用系统**：在内容基础上扩展玩法深度，应用概念层级

3. **设计边界清晰**
   - 明确哪些是世界观概念（概念层级）
   - 明确哪些是核心玩法（顶级层级）
   - 明确哪些是内容支撑（支撑层级）
   - 明确哪些是玩法扩展（应用层级）
   - 避免在设计时混淆层次

4. **概念层级的作用**
   - **提供设计方向**：所有系统设计都要符合概念层级的世界观
   - **保持一致性**：确保游戏设计在文化背景和世界观上保持一致
   - **指导设计灵感**：概念层级为系统设计提供灵感和依据


## 各层级驱动力

游戏的好玩性来自各层级系统提供的驱动力：

1. **概念层级的驱动力**
   - **文化认同感**：玩家对世界观和文化背景的认同
   - **探索欲望**：对未知概念和设定的好奇心
   - **沉浸感**：世界观的一致性增强代入感
   - **驱动力强度**：⭐⭐⭐ (3/5) - 长期驱动力，影响玩家选择游戏

2. **顶级层级的驱动力**
   - **核心玩法驱动**：战斗、任务、探索等核心玩法提供即时反馈
   - **成长驱动**：能力提升、等级成长提供长期目标
   - **选择驱动**：玩家的选择影响游戏世界，提供决策意义
   - **驱动力强度**：⭐⭐⭐⭐⭐ (5/5) - 最强驱动力，是游戏的核心

3. **支撑层级的驱动力**
   - **内容驱动**：丰富的道具、装备、剧情提供收集和探索乐趣
   - **发现驱动**：发现新内容、新剧情提供惊喜感
   - **驱动力强度**：⭐⭐⭐⭐ (4/5) - 重要驱动力，提供内容深度

4. **应用层级的驱动力**
   - **深度驱动**：合成、强化等机制提供玩法深度
   - **策略驱动**：组合、搭配提供策略思考空间
   - **驱动力强度**：⭐⭐⭐ (3/5) - 扩展驱动力，增加玩法深度


##·# 取舍决策示例

**场景**：资源有限，需要决定装备系统的复杂度

**驱动力分析**：
- 装备系统（支撑层级）：⭐⭐⭐⭐ (4/5) - 重要
- 装备强化（应用层级）：⭐⭐⭐ (3/5) - 扩展

**取舍决策**：
- ✅ **必须**：装备系统基础功能（支撑层级，4星）
- ✅ **优先**：装备强化系统（应用层级，3星，但高价值）
- ❌ **延后**：装备染色系统（应用层级，3星，低价值）

**原则**：优先保证高驱动力层级，应用层级按价值取舍
